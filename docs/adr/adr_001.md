# ADR-001: LangGraph for Orchestration

## Context
We need to coordinate multiple agentic nodes (Orchestrator, Web Researcher, Financial Data, News Agent, Writer, and Critic) in a structured workflow. The system has cyclic paths (e.g., Writer -> Critic -> Writer if revision is needed) and requires checkpointing for Human-in-the-Loop (HITL) approval steps where the execution pauses and resumes based on human action.

## Decision
We chose **LangGraph** as our core orchestration framework.

## Consequences
- **Positive**: 
  - Allows drawing the workflow as a state graph with explicit node and edge definitions.
  - Native support for execution interrupts and state persistent saving, making human-in-the-loop implementation straightforward.
  - Supports cycles, allowing the Critic to easily redirect control back to the Writer with specific instructions.
  - Aligns with modern production GenAI requirements in industry.
- **Negative**:
  - Requires defining state schemas and explicitly managing state transitions, which introduces more boilerplate code compared to simpler sequential frameworks like CrewAI or basic LangChain chains.

## Alternatives Considered
- **CrewAI**: Considered for its quick agent setup. However, it lacks precise control over cyclic execution and custom routing conditions, making complex HITL loops difficult to enforce.
- **AutoGen**: Considered for conversational agent patterns. However, AutoGen's transition logic is less deterministic, which conflicts with our requirement for a clean, audit-friendly research pipeline.
